# ServoLib - Приклади використання

Ця папка містить практичні приклади використання бібліотеки ServoLib для керування DC сервоприводами.

## Доступні приклади

### 1. Керування двигуном через PWM + DIR

**Файли:**
- [MOTOR_PWM_DIR_GUIDE.md](MOTOR_PWM_DIR_GUIDE.md) - Повний посібник
- [motor_pwm_dir_example.c](motor_pwm_dir_example.c) - Приклад коду

**Для кого:**
Ідеально підходить для драйверів типу TB6612FNG, DRV8833 (в режимі PWM/DIR), L293D, L9110S.

**Що демонструє:**
- Ініціалізація драйвера з одним PWM каналом та одним GPIO для напрямку
- Керування швидкістю (0-100%)
- Зміна напрямку обертання
- Плавний розгін та гальмування
- Отримання статистики роботи

**Підключення:**
- 1 PWM канал (PA6) - швидкість
- 1 GPIO (PA0) - напрямок (DIR)

### 2. Магнітний енкодер AEAT-9922

**Файли:**
- [AEAT9922_QUICK_START.md](AEAT9922_QUICK_START.md) - Швидкий старт
- [aeat9922_encoder_example.c](aeat9922_encoder_example.c) - Приклад коду
- [../Doc/AEAT-9922_DRIVER_GUIDE.md](../Doc/AEAT-9922_DRIVER_GUIDE.md) - Повна інструкція

**Для кого:**
Високоточний абсолютний магнітний енкодер від Broadcom з SPI інтерфейсом.

**Що демонструє:**
- Ініціалізація SPI та енкодера
- Зчитування абсолютної позиції (0-360°)
- Обчислення швидкості обертання
- Калібрування точності та нульової точки
- Діагностика стану магніту
- Програмування EEPROM

**Підключення:**
- SPI (PA5, PA6, PA7, PA4) - зв'язок з енкодером
- GPIO (PB0) - MSEL для вибору режиму
- Опціонально: TIM2 (PA0, PA1) для інкрементального виходу

**Характеристики:**
- Роздільна здатність: 10-18 біт (до 262144 позицій)
- Точність: ±0.1° @ 25°C
- Інтерфейс: SPI до 10 MHz
- Одночасно абсолютний + інкрементальний вихід

### 3. Керування двигуном через Dual PWM (застарілий приклад)

**Файли:**
- [../MOTOR_DRIVER_EXAMPLE.md](../MOTOR_DRIVER_EXAMPLE.md) - Застарілий приклад з HAL

**Примітка:** Цей приклад використовує застарілу архітектуру HAL та потребує оновлення до HWD.

**Для кого:**
Для драйверів типу L298N, H-bridge з двома PWM входами.

**Підключення:**
- 2 PWM канали (PA6, PA7) - прямий та зворотний хід

## Як використовувати приклади

### Крок 1: Налаштування CubeMX

Налаштуйте периферію згідно з інструкціями в відповідному посібнику:
- PWM таймери (TIM3)
- GPIO піни
- I2C (опціонально, для датчиків)

### Крок 2: Додавання ServoLib до проекту

**Include paths** (Project Properties → C/C++ Build → Settings):
```
../ServoLib/Inc
../ServoLib/Board/STM32F411
```

**Source files:**
```
ServoLib/Src/**/*.c
ServoLib/Board/STM32F411/*.c
```

### Крок 3: Копіювання прикладу

Скопіюйте відповідний файл `.c` з прикладом до вашого проекту або використайте код безпосередньо в `main.c`.

### Крок 4: Компіляція та тестування

1. Скомпілюйте проект
2. Завантажте на STM32
3. Підключіть мотор через драйвер
4. Запустіте тести

## Структура прикладів

Кожен приклад містить:

1. **Ініціалізацію** - Налаштування HWD та драйвера
2. **Базові тести** - Прості функції для перевірки
3. **Розширені приклади** - Складніші сценарії
4. **Обробку помилок** - Як працювати з помилками
5. **Документацію** - Детальні пояснення

## Рекомендації

### Для початківців

Починайте з **PWM + DIR** прикладу:
- Простіше підключення (2 піни)
- Зрозуміліша логіка
- Менше коду

### Для досвідчених

Після освоєння базових прикладів, переходьте до:
- Інтеграції з датчиками положення (AS5600)
- Налаштування PID регуляторів
- Використання повного Servo Controller
- Додавання електронних гальм

## Безпека

**ЗАВЖДИ:**
- Починайте з низької потужності (10-20%)
- Зупиняйте мотор перед зміною напрямку
- Використовуйте зовнішнє живлення для мотора
- З'єднуйте GND STM32 та GND джерела мотора
- Закріплюйте мотор надійно перед тестами

**НІКОЛИ:**
- Не запускайте на повну потужність без навантаження
- Не міняйте напрямок без зупинки
- Не перевищуйте максимальний струм драйвера
- Не залишайте мотор працювати без нагляду

## Підтримка

При виникненні питань:
1. Прочитайте повну документацію в відповідному `.md` файлі
2. Перевірте схему підключення
3. Використайте осцилоскоп для діагностики PWM
4. Перегляньте налаштування CubeMX
