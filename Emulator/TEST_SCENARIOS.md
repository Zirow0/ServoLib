# Приклади команд для тестування UDP емуляції

Цей документ містить приклади команд та сценаріїв для тестування системи UDP емуляції сервоприводу.

## Тестові сценарії

### 1. Простий тест підключення

**Мета**: Перевірити зв'язок між емулятором та моделлю

**Кроки**:
1. Запустити математичну модель двигуна
2. Запустити емулятор ServoLib
3. Перевірити повідомлення про успішне підключення
4. Відстежити обмін даними протягом 10 секунд

**Очікуваний результат**:
- Встановлення UDP з'єднання
- Регулярний обмін даними (кожні 1 мс)
- Відсутність помилок зв'язку

### 2. Тест керування двигуном

**Мета**: Перевірити команди керування двигуном

**Кроки**:
1. Керування за допомогою команд:
   - `Servo_SetPosition(&servo, 90.0f)` - переміщення до 90°
   - `Servo_SetVelocity(&servo, 50.0f)` - встановлення швидкості 50 град/с
   - `Servo_SetPower(&motor, 25.0f)` - 25% потужності вперед
   - `Servo_SetPower(&motor, -25.0f)` - 25% потужності назад

2. Спостереження за:
   - Зміною позиції двигуна
   - Зміною швидкості
   - Поточним струмом
   - Станом двигуна

**Очікуваний результат**:
- Плавна зміна позиції відповідно до команд
- Реалістична динаміка (прискорення/гальмування)
- Правильний напрямок обертання
- Адекватні значення струму

### 3. Тест системи гальм

**Мета**: Перевірити функціональність системи гальм

**Кроки**:
1. Керування гальмами:
   - `Brake_Release(&brake)` - відпустити гальма
   - `Brake_Engage(&brake)` - активувати гальма
   - `Brake_EmergencyEngage(&brake)` - аварійне гальмування

2. Спостереження за:
   - Станом гальм (активні/відпущені)
   - Впливом на двигун (блокування обертання)
   - Часом реакції гальм

**Очікуваний результат**:
- Гальма блокують рух при активації
- Двигун не може обертатися при активних гальмах
- Після відпускання гальм двигун може рухатися

### 4. Тест сенсора положення

**Мета**: Перевірити отримання даних з сенсора

**Кроки**:
1. Керування двигуном і одночасне зчитування даних:
   ```c
   float angle, velocity;
   Sensor_UDP_ReadAngle(&sensor, &angle);
   Sensor_UDP_GetVelocity(&sensor, &velocity);
   ```

2. Спостереження за:
   - Точністю показів сенсора
   - Затримками між рухом і оновленням даних
   - Стабільністю показів

**Очікуваний результат**:
- Дані сенсора відповідають позиції двигуна
- Мінімальні затримки оновлення
- Стабільні значення без надмірного шуму

### 5. Комплексний тест руху

**Мета**: Перевірити роботу всіх компонентів разом

**Кроки**:
1. Команди послідовно:
   - Відпустити гальма
   - Встановити цільову позицію 180°
   - Чекати досягнення цілі
   - Зупинити систему
   - Активувати гальма

2. Моніторинг:
   - Позиції та швидкості протягом руху
   - Струму двигуна
   - Стану всіх компонентів
   - Часу виконання операції

**Очікуваний результат**:
- Плавний рух до цільової позиції
- Правильне відстеження досягнення цілі
- Адекватна поведінка системи захисту
- Без помилок під час операції

### 6. Тест аварійної ситуації

**Мета**: Перевірити реакцію системи на аварійні ситуації

**Кроки**:
1. Імітація аварійної ситуації:
   - `Servo_EmergencyStop(&servo)` - аварійна зупинка
   - `Brake_EmergencyEngage(&brake)` - аварійне гальмування

2. Спостереження за:
   - Часом зупинки двигуна
   - Активацією гальм
   - Станом системи після аварії
   - Можливістю відновлення

**Очікуваний результат**:
- Негайна зупинка двигуна (< 10 мс)
- Активація гальм для безпечного блокування
- Система переходить у аварійний стан
- Потребує ініціалізації для подальшої роботи

### 7. Тест стійкості до втрати зв'язку

**Мета**: Перевірити поведінку при втраті зв'язку з моделлю

**Кроки**:
1. Запустити нормальний процес
2. Тимчасово відключити модель двигуна
3. Спостереження за поведінкою емулятора:
   - Кількість таймаутів
   - Поведінка при втраті зв'язку
   - Спроби повторного підключення

4. Відновити зв'язок та продовжити роботу

**Очікуваний результат**:
- Система виявляє втрату зв'язку
- Використовує попередні значення або безпечний стан
- Відновлює зв'язок при поверненні моделі
- Продовжує нормально працювати

### 8. Тест продуктивності

**Мета**: Перевірити продуктивність системи

**Кроки**:
1. Запустити тривалий тест (5-10 хвилин):
   - Неперервне керування
   - Регулярне зчитування даних
   - Відстеження всіх параметрів

2. Заміряти:
   - Частоту оновлення
   - Кількість втрачених пакетів
   - Використання процесора
   - Використання пам'яті

**Очікуваний результат**:
- 1000 Гц частота оновлення
- Мінімальна кількість втрат
- Стабільне використання ресурсів
- Відсутність витоку пам'яті

## Приклади тестових функцій

### Простий тест руху
```c
void Test_Motion(void) {
    printf("Starting motion test...\n");
    
    // Активувати гальма на початку
    Brake_Engage(&brake);
    HWD_Timer_DelayMs(200);  // Чекати
    
    // Відпустити гальма
    Brake_Release(&brake);
    HWD_Timer_DelayMs(100);  // Зачекати перед рухом
    
    // Встановити цільову позицію
    Servo_SetPosition(&servo, 90.0f);
    
    // Основний цикл
    for (int i = 0; i < 1000; i++) {  // 1 секунда при 1 кГц
        Servo_Update(&servo);
        Brake_Update(&brake);
        
        if (Servo_IsAtTarget(&servo)) {
            printf("Target reached at %d ms\n", i);
            break;
        }
        
        HWD_Timer_DelayMs(1);
    }
    
    // Зупинити та активувати гальма
    Servo_Stop(&servo);
    Brake_Engage(&brake);
    printf("Motion test completed\n");
}
```

### Тест зміни напрямку
```c
void Test_DirectionChange(void) {
    printf("Starting direction change test...\n");
    
    Brake_Release(&brake);
    
    // Рух в одному напрямку
    Servo_SetPower(&motor, 30.0f);  // 30% вперед
    for (int i = 0; i < 500; i++) {  // 0.5 секунди
        Servo_Update(&servo);
        HWD_Timer_DelayMs(1);
    }
    
    // Зміна напрямку
    Servo_SetPower(&motor, -30.0f); // 30% назад
    for (int i = 0; i < 500; i++) {  // 0.5 секунди
        Servo_Update(&servo);
        HWD_Timer_DelayMs(1);
    }
    
    Servo_Stop(&servo);
    Brake_Engage(&brake);
    printf("Direction change test completed\n");
}
```

Ці тестові сценарії допоможуть повністю перевірити функціональність системи UDP емуляції сервоприводу.
