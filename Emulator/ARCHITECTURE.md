Цей документ описує архітектуру взаємодії між емулятором ServoLib та математичною моделлю двигуна через UDP протокол.

## Детальна архітектура взаємодії

### 1. Емулятор ServoLib

#### 1.1. Рівень керування (Ctrl/)
- **Servo Controller**: Основна логіка керування, використовує отримані дані
- **PID Controller**: Розраховує команди на основі даних з сенсора
- **Safety System**: Контролює обмеження та аварійні ситуації
- **Trajectory Generator**: Генерує траєкторії руху

#### 1.2. Апаратна абстракція (HWD/)
- **HWD_UDP**: Універсальний інтерфейс для UDP комунікації
- **Message Structure**: Стандартизована структура повідомлень
- **Sequence Management**: Управління номерами послідовностей
- **Timeout Handling**: Обробка таймаутів та повторних відправок

#### 1.3. Драйвери компонентів
- **Brake UDP Driver**: Керує системою гальм, обробляє команди/стани
- **Motor UDP Driver**: Керує двигуном, передає команди потужності
- **Sensor UDP Driver**: Запитує та отримує дані сенсора

### 2. Комунікаційний рівень

#### 2.1. UDP Протокол
- **Порт 8889**: Локальний порт емулятора
- **Порт 8888**: Порт сервера моделі
- **Таймаут**: 100 мс на отримання відповіді
- **Частота оновлення**: 1000 Гц (1 мс)

#### 2.2. Структура повідомлень
- **Header**: Тип повідомлення, номер послідовності
- **Payload**: Дані конкретного типу
- **Checksum**: (За потреби) для перевірки цілісності

### 3. Математична модель

#### 3.1. UDP Сервер
- **Прослуховування**: Порт 8888
- **Обробка пакетів**: Розбір отриманих команд
- **Відповіді**: Генерація та відправка статусів

#### 3.2. Фізична модель двигуна
- **Динаміка**: Розрахунки на основі фізичних законів
- **Електричні параметри**: Струм, напруга, ЕРС
- **Механічні параметри**: Позиція, швидкість, прискорення
- **Моделювання помилок**: Заклинювання, перевищення струму

#### 3.3. Симуляція компонентів
- **Сенсорна модель**: Емілює поведінку енкодера
- **Гальмова модель**: Емілює механіку гальм
- **Шум і затримки**: Додавання реалістичних властивостей

## Потоки даних

### 1. Потік команд (Емулятор → Модель)
```
Servo Controller → Motor Driver → UDP_Send(MOTOR_CMD) → Model
Servo Controller → Brake Driver → UDP_Send(BRAKE_CMD) → Model
Sensor Driver → UDP_Send(SENSOR_CMD) → Model
```

### 2. Потік статусів (Модель → Емулятор)
```
Model → UDP_Send(MOTOR_STATE) → Motor Driver → Servo Controller
Model → UDP_Send(BRAKE_STATE) → Brake Driver → Servo Controller
Model → UDP_Send(SENSOR_STATE) → Sensor Driver → Servo Controller
```
